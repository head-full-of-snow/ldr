问题一：配置缺失 doi_resolvers

KeyError

现象

KeyError: 'default_doi_resolver'
File: searx/preferences.py:445
  MultipleChoiceSetting(
    [settings['default_doi_resolver'], ],  ← KeyError
    choices=DOI_RESOLVERS                   ← DOI_RESOLVERS = list(settings['doi_resolvers'])
  )
修复

根因

SimpleXNG v0.1.3 升级上游 SearXNG 到 2025.8.3，新增了 default_doi_resolver 和 doi_resolvers 配置项。旧配置文件由旧版本生成，不含这两个字段，导致 KeyError。且 DOI_RESOLVERS 变量定义为 list(settings['doi_resolvers'])，依赖配置文件中的 doi_resolvers 字典。

修复方法

在 ~/.config/simplexng/simplexng_settings.yml 中添加：

doi_resolvers:
  oadoi.org: 'https://oadoi.org/'
  doi.org: 'https://doi.org/'
  sci-hub.se: 'https://sci-hub.se/'
  sci-hub.st: 'https://sci-hub.st/'
  sci-hub.ru: 'https://sci-hub.ru/'

default_doi_resolver: "oadoi.org"

验证

配置文件加载后 settings['doi_resolvers'] 返回包含 5 个 key 的字典，settings['default_doi_resolver'] 值为 oadoi.org。

问题二：依赖缺失 sniffio

ImportError

现象

simplexng --open --port 6464
ERROR: ModuleNotFoundError: No module named 'sniffio'
修复

根因

SimpleXNG 的 PyPI 包缺少对 sniffio 的依赖声明。该包被 anyio（httpx 的依赖）间接需要，但未在 pyproject.toml 中显式声明。

修复方法

使用 uv 工具重新安装时注入缺失依赖：

uv tool install simplexng --with sniffio --reinstall

验证

安装日志中出现 + sniffio==1.3.1，标准 simplexng 命令从此正常工作。

问题三：CLI 默认值硬编码

行为不符合预期

现象

simplexng 不带参数启动时：

  • 端口默认 8888（而非配置文件的 16464）
  • 绑定地址默认 127.0.0.1（而非配置文件的 0.0.0.0）
  • 先前的临时方案：每次必须加 --port 6464 -H 0.0.0.0
修复

根因

simplexng/simplexng.py 的 create_parser() 函数将端口和主机默认值硬编码：

# 修改前：
parser.add_argument("-p", "--port", type=int, default=8888, ...)
parser.add_argument("-H", "--host", default="127.0.0.1", ...)

配置文件的 server.port 和 server.bind_address 仅在 新建 配置文件时使用，启动服务器始终使用 CLI 默认值。

修复方法

修改 simplexng.py 的 create_parser()，使其在解析 CLI 参数前尝试读取配置文件：

def create_parser() -> argparse.ArgumentParser:
    default_port = 8888
    default_host = "127.0.0.1"

    try:
        import yaml
        _config_path = get_settings_dir(CFG_NAME) / _SETTINGS_NAME
        if _config_path.exists():
            _cfg = yaml.safe_load(_config_path.read_text())
            if _cfg and 'server' in _cfg:
                default_port = _cfg['server'].get('port', default_port)
                default_host = _cfg['server'].get('bind_address', default_host)
    except Exception:
        pass

    # CLI 参数仍可覆盖配置文件：
    parser.add_argument("-p", default=default_port, ...)
    parser.add_argument("-H", default=default_host, ...)

同时添加了必要的 import：

from simplexng.settings import ..., get_settings_dir, CFG_NAME
from simplexng.settings import SETTINGS_NAME as _SETTINGS_NAME

验证

simplexng --help
  -p, --port PORT       Port to run on (default: 16464)
  -H, --host HOST       Host to bind to (default: 0.0.0.0)

不加参数直接启动：监听 0.0.0.0:16464，HTTP 200 ✓

问题四：旧 Cookie 语言设置验证失败

ValidationException

现象

首次访问页面时连续抛两次错误：

ERROR: Invalid value: "zh-CN"
preferences.py:90 → ValidationException
preferences.py:462 → parse_dict(sxng_request.cookies)

虽然异常被 webapp.py 捕获，但请求处理提前终止，影响用户体验。

修复

根因

浏览器存储了旧版本 SearXNG 的 locale=zh-CN Cookie。当前版本的 LOCALE_NAMES（从 data/locales.json 加载）中不包含 zh-CN。EnumStringSetting._validate_selection() 对任何不在 self.choices 的值直接抛 ValidationException，没有降级机制。

修复方法

修改 searx/preferences.py 中三个 Setting 类的 parse() 方法，在值不合法时静默保留默认值（self.value）：

1. EnumStringSetting.parse()

def parse(self, data: str):
    try:
        self._validate_selection(data)
        self.value = data
    except ValidationException:
        # 保留默认值，不抛异常
        self.value = self.value

2. MultipleChoiceSetting.parse() — 同理添加 try/except：

def parse(self, data: str):
    self.value = self.parse_csv(data)
    try:
        self._validate_selections(self.value)
    except ValidationException:
        self.value = self.default_value  # ← 实际代码中使用 self.value

3. MapSetting.parse() — 在 key 不合法时 fallback：

def parse(self, data: str):
    if data not in self.map:
        self.value = self.value  # keep default
        # 找到默认值对应的 key
        self.key = list(self.map.keys())[list(self.map.values()).index(self.value)]
    else:
        self.value = self.map[data]
        self.key = data
