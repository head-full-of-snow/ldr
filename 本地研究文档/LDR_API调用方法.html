2 方案一：REST API v1 直接调用

测试结论：REST API v1 的 /api/v1/quick_summary 端点可直接通过 curl 调用，无需 CSRF token，仅需登录 session cookie。返回完整研究结果，包含摘要、来源、迭代详情。

2.1 环境信息

端点认证方式CSRF说明
/api/v1/health无需认证不需要健康检查
/api/v1/quick_summarySession Cookie不需要简易研究
/api/v1/generate_reportSession Cookie不需要完整报告
/api/v1/analyze_documentsSession Cookie不需要文档分析

2.2 完整调用流程

Step 1：登录获取 Session Cookie

# 获取登录页面和 CSRF token（用于表单登录）
curl -s -c cookie.txt http://10.10.20.21:5000/auth/login \
  | python3 -c "import sys,re; \
    m=re.search(r'name=\"csrf_token\" value=\"([^\"]+)\"', sys.stdin.read()); \
    print(m.group(1))" > csrf.txt

# 提交登录
curl -s -b cookie.txt -c cookie.txt \
  -X POST http://10.10.20.21:5000/auth/login \
  -d "username=zxh&password=@zxh19859769907&csrf_token=$(cat csrf.txt)"
# 返回 302 重定向表示登录成功

Step 2：简易研究（Quick Summary）

# 调用 quick_summary - 同步返回完整结果
curl -s -b cookie.txt \
  -X POST http://10.10.20.21:5000/api/v1/quick_summary \
  -H "Content-Type: application/json" \
  -d '{
    "query": "你想研究的问题",
    "iterations": 2,
    "temperature": 0.7
  }'
实测返回字段： research_id（研究ID）, summary（摘要文本）, findings（分阶段发现）, sources（来源列表）, iterations（迭代次数）, formatted_findings（格式化全文）

Step 3：完整报告（Generate Report）

# 调用 generate_report - 生成 Markdown 格式报告
curl -s -b cookie.txt \
  -X POST http://10.10.20.21:5000/api/v1/generate_report \
  -H "Content-Type: application/json" \
  -d '{
    "query": "你想研究的主题",
    "output_file": "/path/to/report.md",
    "searches_per_section": 2,
    "temperature": 0.5
  }'

Step 4：健康检查（无需登录）

curl -s http://10.10.20.21:5000/api/v1/health
# 返回: {"status":"ok","message":"API is running","timestamp":...}

3 方案二：Web UI API（异步研究）

测试结论：Web API 的 /api/start_research 可启动异步研究任务，需要 CSRF token。研究在后台执行，可通过轮询获取状态和报告。

3.1 完整调用流程

# Step 1: 登录（同上）

# Step 2: 获取 CSRF Token
curl -s -b cookie.txt http://10.10.20.21:5000/auth/csrf-token
# 返回: {"csrf_token":"ImQ1ZGNhMWFj..."}

# Step 3: 启动研究
curl -s -b cookie.txt \
  -X POST http://10.10.20.21:5000/api/start_research \
  -H "Content-Type: application/json" \
  -H "X-CSRFToken: <上面获取的csrf_token>" \
  -d '{"query": "研究问题", "research_mode": "quick"}'
# 返回: {"research_id": "uuid", "status": "success"}

# Step 4: 查询研究状态
curl -s -b cookie.txt http://10.10.20.21:5000/api/research/<research_id>/status

# Step 5: 获取研究报告
curl -s -b cookie.txt http://10.10.20.21:5000/api/report/<research_id>

4 方案三：Python 直接调用（无需 Web 服务）

测试结论：通过 SSH 连接到服务器，在 conda 环境中直接 import 调用 quick_summary 成功。该方案不依赖 5000 端口 Web 服务，但需要服务器 Python 环境。generate_report 因耗时较长超时，但函数签名和初始化均正常。

4.1 执行流程

quick_summary("查询内容") │ ├─ ① @no_db_settings 装饰器 → 禁用数据库设置读取 │ ├─ ② create_settings_snapshot() → 构建 in-memory 配置 │ ├─ JSON 默认文件 (defaults/default_settings.json) │ └─ 环境变量覆盖 (LDR_* 前缀) │ ├─ ③ _init_search_system() │ ├─ get_llm() → 创建 LangChain LLM 实例 │ ├─ get_search() → 创建搜索引擎实例 │ └─ AdvancedSearchSystem → 研究引擎 │ ├─ ④ system.analyze_topic(query) → 执行研究 │ └─ ⑤ 返回 {"summary", "findings", "sources", ...}

4.2 设置读取优先级

优先级来源说明
1 (最高)函数参数provider=, api_key=, settings_override={}
2环境变量LDR_LLM_MODEL, LDR_LLM_PROVIDER 等
3JSON 默认文件defaults/default_settings.json
4数据库被 @no_db_settings 禁用

8 注意事项与常见问题

1. 必须配置模型
Python 直接调用时，llm.model 默认为空字符串，不配置会抛出 ValueError: LLM model not configured。
解决：通过 settings_override={"llm.model": "MiniMax-M2.7"} 或设置环境变量 LDR_LLM_MODEL。
2. 搜索引擎依赖
默认搜索引擎为 searxng，需要运行 SearXNG 实例。如果搜索服务不可用，研究仍能运行但搜索结果为空。
可通过 settings_override={"search.tool": "wikipedia"} 切换到 Wikipedia 搜索（无需额外服务）。
3. generate_report 耗时较长
完整报告生成涉及多轮搜索和综合，通常需要数分钟。建议在异步任务或 tmux 会话中运行。
