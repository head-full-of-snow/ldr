## 一、启动方式

### 方式 1：tmux 一键启动（推荐）

```bash
cd /kanas/datav/lmet/projects/local-deep-research
bash quick_start.sh                  # 默认端口 5000
bash quick_start.sh -p 8080          # 指定端口
bash quick_start.sh --stop           # 停止
bash quick_start.sh --restart        # 重启
```

启动后通过 `tmux attach -t ldr` 进入会话，`Ctrl+B` 然后按 `D` 退出会话（服务继续运行）。

### 方式 2：直接启动

```bash
conda activate ldr
ldr-web
```

前台运行，`Ctrl+C` 停止。

### 方式 3：nohup 后台启动

```bash
conda activate ldr
nohup ldr-web > /tmp/ldr.log 2>&1 &
```

查看日志：`tail -f /tmp/ldr.log`
停止服务：`pkill -f ldr-web`

### 方式 4：完整 Python 路径（远程非交互式 SSH 场景）

当 `conda activate` 在非交互式 shell 中不生效时：

```bash
nohup /data/miniforge/envs/ldr/bin/python -m local_deep_research.web.app > /tmp/ldr.log 2>&1 &
```

## 二、自定义端口和监听地址

### 方法 1：环境变量（优先级最高，推荐）

```bash
export LDR_WEB_PORT=8080           # 端口
export LDR_WEB_HOST=127.0.0.1      # 仅本地访问（默认 0.0.0.0 对外开放）
export LDR_APP_DEBUG=true           # 调试模式
ldr-web
```

也可以写在一行：

```bash
LDR_WEB_PORT=8080 ldr-web
```

### 方法 2：server_config.json 配置文件

在数据目录下创建配置文件：

```bash
cat > ~/.local/share/local-deep-research/server_config.json << 'EOF'
{
    "host": "0.0.0.0",
    "port": 8080,
    "debug": false,
    "use_https": false,
    "allow_registrations": true
}
EOF
```

优先级：环境变量 > server_config.json > 默认值。

修改后需要重启服务生效。

## 三、大模型（LLM）配置

### Web 界面配置（推荐）

1. 登录后点击右上角 **Settings**（齿轮图标）
2. 选择 **Language Models** 标签页
3. 配置以下项目：

| 设置项 | 说明 |
|--------|------|
| **LLM Provider** | 选择提供商（下拉框） |
| **LLM Model** | 填写模型名称 |
| 对应的 URL 字段 | 填写 API base_url |
| 对应的 API Key 字段 | 填写密钥 |

### 各 Provider 的 URL 字段

| Provider | URL 字段 | 适用场景 |
|----------|----------|----------|
| OpenAI-Compatible Endpoint | OpenAI Endpoint Url | 任意 OpenAI 兼容 API（vLLM、llama.cpp、OpenRouter 等） |
| Ollama | Ollama Endpoint Url | 本地 Ollama 服务 |
| OpenAI | OpenAI Api Base | OpenAI 官方或代理 |
| LM Studio | Lmstudio Url | 本地 LM Studio |
| LlamaCpp | Llamacpp Url | llama-server |

### 环境变量配置（覆盖 Web 设置）

通过环境变量设置，会锁定 Web 界面对应选项为只读：

```bash
export LDR_LLM_PROVIDER=openai_endpoint
export LDR_LLM_MODEL=gpt-4o
export LDR_LLM_OPENAI_ENDPOINT_URL=https://your-api.com/v1
export LDR_LLM_OPENAI_ENDPOINT_API_KEY=sk-xxxx
```

### 配置存储位置

LLM 等所有用户设置存储在**每用户加密数据库**中：

```
~/.local/share/local-deep-research/
├── ldr_auth.db                              # 认证数据库（SQLite，未加密）
├── .secret_key                              # 加密密钥
├── encrypted_databases/                     # 用户加密数据库目录
│   ├── ldr_user_<hash>.db                   # 用户数据（SQLCipher 加密）
│   └── ldr_user_<hash>.db.salt              # 加密盐值
├── research_outputs/                        # 研究输出
├── library/                                 # 文档库
└── logs/                                    # 日志
```

## 四、查看加密数据库

用户数据库使用 SQLCipher (AES-256) 加密，需要通过项目自带的工具或 sqlcipher 命令行访问。

### 方法 1：sqlcipher 命令行

```bash
# 安装 sqlcipher（如未安装）
sudo apt install sqlcipher

# 打开加密数据库（需要用户密码）
# 密码即用户登录密码的派生值，通过 salt 文件和 PBKDF2 生成
sqlcipher ~/.local/share/local-deep-research/encrypted_databases/ldr_user_*.db
```

> 注意：直接用 sqlcipher 打开需要正确的 PRAGMA key，密钥由用户登录密码派生而来，不是明文密码。

### 方法 2：通过 Web API 查看（推荐）

```bash
# 先登录获取 session
curl -s -c /tmp/ldr_cookie http://localhost:5000/auth/login | grep csrf_token

# 用用户名密码登录（替换为实际值）
curl -s -b /tmp/ldr_cookie -c /tmp/ldr_cookie \
  -d "username=YOUR_USER&password=YOUR_PASS&csrf_token=CSRF_TOKEN" \
  -X POST http://localhost:5000/auth/login

# 获取当前设置
curl -s -b /tmp/ldr_cookie http://localhost:5000/settings/api | python3 -m json.tool
```

## 五、常用环境变量速查

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| `LDR_WEB_HOST` | 监听地址 | `0.0.0.0` |
| `LDR_WEB_PORT` | 监听端口 | `5000` |
| `LDR_APP_DEBUG` | 调试模式 | `false` |
| `LDR_DATA_DIR` | 数据存储目录 | `~/.local/share/local-deep-research` |
| `LDR_LLM_PROVIDER` | LLM 提供商 | 无 |
| `LDR_LLM_MODEL` | 模型名称 | 无 |
| `LDR_LLM_OPENAI_ENDPOINT_URL` | 自定义 API 地址 | 无 |
| `LDR_LLM_OPENAI_ENDPOINT_API_KEY` | 自定义 API 密钥 | 无 |
| `LDR_LLM_OPENAI_API_KEY` | OpenAI 密钥 | 无 |
| `LDR_LLM_ANTHROPIC_API_KEY` | Anthropic 密钥 | 无 |
| `LDR_APP_ALLOW_REGISTRATIONS` | 是否允许注册 | `true` |

命名规则：设置项 `llm.openai_endpoint.url` 对应环境变量 `LDR_LLM_OPENAI_ENDPOINT_URL`（将 `.` 替换为 `_` 并大写）。

## 六、注意事项

- 首次访问会跳转到登录/注册页面，需先注册账号
- CUDA 驱动版本较旧（12060），torch 以 CPU 模式运行。如需 GPU 加速，需更新 NVIDIA 驱动
- 环境变量设置会覆盖并锁定 Web 界面中的对应选项（变为只读）
- 修改端口/地址后需重启服务生效
