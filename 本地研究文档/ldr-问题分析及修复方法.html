SearXNG categories=["general"] 触发被阻断的后端引擎

High

实时监听日志证据

14:41:13.336 | INFO | searxng:_get_previews:444 - Getting SearXNG previews for query: algorithmic trading technology 14:41:13.353 | INFO | searxng:_get_search_results:281 - Sending request to SearXNG instance at http://10.10.20.21:16464 14:41:13.385 | INFO | searxng:_get_search_results:314 - Found 1 search result elements 14:41:13.388 | WARNING| searxng:_get_search_results:390 - SearXNG backend engine failed or rate-limited: arxiv 14:41:13.388 | WARNING| searxng:_get_search_results:404 - SearXNG returned no valid results for query: algorithmic trading technology 14:41:13.455 | MILESTONE | research_service - Result from web_search: No results found for 'algorithmic trading technology'

根因

已实测验证：LDR 默认 categories=["general"]，SearXNG 的 general 分类触发 Google/DuckDuckGo 后端（被防火墙阻断）。 而 arxiv/pubmed 属于 science 分类，LDR 根本没请求这个分类。

categories=general (LDR默认)→ Google/DuckDuckGo → BLOCKED → 0 结果
categories=science→ arxiv/pubmed → 20 结果

涉及文件

  • src/local_deep_research/defaults/default_settings.json 第 2725 行: "value": ["general"]
  • src/local_deep_research/web_search_engines/engines/search_engine_searxng.py 第 163 行: or ["general"]

修复

方法 1: Web UI 设置（不改代码，最简单）

http://10.10.20.21:5000/settings
→ 搜索 SearXNG → Categories
→ 将 ["general"] 改为 ["science", "general"]

方法 2: 改配置文件（永久生效）

# 服务器上编辑:
vi /kanas/datav/lmet/projects/local-deep-research/src/local_deep_research/defaults/default_settings.json
# 第 2725 行，将:
"value": ["general"]
# 改为:
"value": ["science", "general"]

方法 3: 改源码兜底值

# search_engine_searxng.py 第 163 行
self.categories = self._normalize_list(categories) or ["science", "general"]

Playwright Chromium 浏览器未安装 — 内容抓取全部失败

Critical — 新发现

实时监听日志证据

14:42:37.982 | INFO | playwright_html - Auto: no/short content for https://www.bridgewater.com/..., falling back to JS rendering 14:42:39.157 | ERROR | playwright_html:_fetch_with_playwright:271 - Playwright error fetching https://www.bridgewater.com/... playwright._impl._errors.Error: BrowserType.launch: Executable doesn't exist at /home/zengxinghang/.cache/ms-playwright/chromium_headless_shell-1217/chrome-headless-shell-linux64/chrome-headless-shell ┌──────────────────────────────────────────────────────────────┐ │ Looks like Playwright was just installed or updated. │ │ Please run the following command to download new browsers: │ │ │ │ playwright install │ └──────────────────────────────────────────────────────────────┘

根因

  • Playwright Python 包已安装，但 Chromium 浏览器二进制文件未下载
  • 路径 ~/.cache/ms-playwright/chromium_headless_shell-1217/ 不存在
  • 推测是 Playwright 更新后未运行 playwright install
  • 当前有 18 个僵尸 Playwright 进程来自旧版本，新版本从未成功启动过

影响

所有需要 JS 渲染的网页内容抓取全部失败。LDR 内容抓取链路：HTTP fetch → 失败 → Playwright fallback → 也失败 → 无内容。

为什么需要 Playwright？

LDR 内容抓取两阶段机制

阶段 1：HTTP 直接抓取（用 requests/trafilatura）— 速度快，但遇到 CAPTCHA、JS 渲染页面、付费墙就失败。

阶段 2：Playwright 兜底（用无头 Chromium 浏览器）— 像真人一样打开网页，等 JS 执行完再提取内容。可以绕过大部分 CAPTCHA 和 JS 渲染问题。

当前 Chromium 未安装，阶段 2 完全不可用，遇到反爬的网站就彻底抓不到内容。

为什么 playwright install chromium 不成功？

Playwright 1.59.0 的 Chromium 下载源均不可用

  • 微软官方 CDN（playwright.azureedge.net）→ 重定向到 playwright.download.prss.microsoft.com → 400 Bad Request
  • 中国镜像（registry.npmmirror.com）→ 404 NOT_FOUND（v1217 版本尚未同步）
  • Playwright 进程卡死：反复重试下载，导致 SSH 连接超时

修复方案

方案 A：从 Google 官方源手动下载（已验证可用，推荐）

# 1. 清理旧版僵尸进程
pkill -f 'playwright/driver/package/cli.js run-driver'

# 2. 创建目标目录
mkdir -p ~/.cache/ms-playwright/chromium-1217
cd ~/.cache/ms-playwright/chromium-1217

# 3. 从 Google Chrome for Testing 下载（服务器可访问此源，已验证 HTTP 200）
curl -L --max-time 600 -o /tmp/chrome-linux64.zip \
  'https://storage.googleapis.com/chrome-for-testing-public/147.0.7727.15/linux64/chrome-linux64.zip'

# 4. 解压
unzip -o /tmp/chrome-linux64.zip
chmod +x chrome-linux64/chrome

# 5. 安装 headless shell（Playwright 实际调用的是这个）
mkdir -p ~/.cache/ms-playwright/chromium_headless_shell-1217
cd ~/.cache/ms-playwright/chromium_headless_shell-1217
curl -L --max-time 300 -o /tmp/chrome-headless-shell.zip \
  'https://storage.googleapis.com/chrome-for-testing-public/147.0.7727.15/linux64/chrome-headless-shell-linux64.zip'
unzip -o /tmp/chrome-headless-shell.zip
chmod +x chrome-headless-shell-linux64/chrome-headless-shell

# 6. 验证
ls -la ~/.cache/ms-playwright/chromium_headless_shell-1217/chrome-headless-shell-linux64/chrome-headless-shell

注意：Google 源下载速度约 29KB/s（170MB 约需 100 分钟），下载已在后台运行中。

方案 B：降级 Playwright 到镜像源有同步的版本

conda activate ldr
pip install playwright==1.52.0   # 降级到有镜像的旧版本
PLAYWRIGHT_DOWNLOAD_HOST=https://registry.npmmirror.com/-/binary/playwright playwright install chromium

方案 C：用系统包安装 Chromium（需 sudo 权限）

sudo apt-get install -y chromium-browser
# 然后设置环境变量让 Playwright 使用系统 Chromium:
export PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH=/usr/bin/chromium-browser

分析

LDR 的内容抓取工具（content_fetcher）试图抓取搜索结果链接的全文，但：

  • 很多金融网站有 CAPTCHA/付费墙 防护，HTTP 直接抓取失败
  • Playwright 兜底（JS 渲染绕过部分防护）也因 Chromium 未安装而失败
  • 导致大量无效的抓取尝试，浪费时间

修复

安装 Playwright Chromium 后（问题 #2 修复），JS 渲染可以绕过部分 CAPTCHA
