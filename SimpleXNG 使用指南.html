一、基本使用

1.1 启动服务

simplexng

默认监听 0.0.0.0:16464，浏览器访问 http://<服务器IP>:16464/ 即可。

1.2 CLI 参数

参数默认值说明
-p, --port从配置读取指定端口
-H, --host从配置读取绑定地址（0.0.0.0 允许外部访问）
--open-自动打开浏览器
--flask-用 Flask 开发服务器（调试用）
--settings-指定自定义配置文件路径
-v, --verbose-详细日志输出
# 常用变体 simplexng --port 8080 -H 127.0.0.1 # 仅本地访问 simplexng --open # 启动 + 打开浏览器 simplexng -p 6464 -H 0.0.0.0 # 外网可访问

1.3 配置文件位置

~/.config/simplexng/simplexng_settings.yml

二、搜索语法

2.1 引擎快捷指令（Bang）

在搜索关键词前加上 !引擎名 可直接指定搜索引擎：

快捷指令引擎示例
!goGoogle!go 芒果种植
!brBrave!br machine learning
!biBing!bi news today
!spStartpage!sp privacy
!arxarxiv!arx quantum computing
!pubPubMed!pub crispr
!seSemantic Scholar!se deep learning
!gosGoogle Scholar!gos climate change

2.2 多引擎搜索

!arx+pub crispr gene editing     # 同时在 arxiv 和 PubMed 搜索
!go+br 今日热点                  # 同时在 Google 和 Brave 搜索

2.3 排除引擎

python -!google                   # 搜索 python，排除 Google 结果

2.4 类别筛选

网页端可通过顶部标签页切换类别，URL 参数方式：

http://<host>:16464/search?q=quantum&categories=science
http://<host>:16464/search?q=cat&categories=images,videos

2.5 API 调用（JSON 格式）

curl 'http://<host>:16464/search?q=test&format=json'

注意：需要配置中 formats 包含 json，且 public_instance: true 才允许远程 API 调用。

3.2 关于 DuckDuckGo

已禁用 原因： 该服务器 IP 被 DuckDuckGo 屏蔽，请求始终超时。 
curl https://html.duckduckgo.com/ → FAIL (ConnectTimeout)
如需要在其他网络环境使用，在配置中将 duckduckgo 的 disabled: true 改为 false。

3.3 配置文件的 use_default_settings

当前配置为 use_default_settings: false，意为只加载配置文件中显式列出的引擎。

若改为 true，SearXNG 内置的 100+ 引擎（Wikipedia、GitHub、StackExchange 等）将全部生效。适合需要更多引擎但不想逐一配置的场景。

五、常见问题与故障排查

5.1 引擎超时 / 无法连接

常见 
ERROR: engine timeout
ERROR: HTTP requests timeout (ConnectTimeout)
WARNING: Task queue depth is 1

原因： 搜索引擎（如 DuckDuckGo）从某些网络环境（尤其是中国服务器）无法直接访问，或响应缓慢导致超时。

解决方案：

  • 禁用不可达的引擎（disabled: true），启用替代引擎（Google、Brave）
  • 适当增加 timeout 值（已设为 6-10 秒）
  • 使用 !引擎名 语法手动选择可达的引擎
  • 如需要访问被屏蔽的引擎，可配置代理：
outgoing:
  proxies:
    all://:
      - http://your-proxy:8080

5.2 Bing 重定向错误

已知 
count_error: 1 redirects, maximum: 0

原因： Bing 检测到非浏览器请求后重定向到 cn.bing.com，但 Bing 引擎的 soft_max_redirects 为 0，不允许重定向。

影响： 部分请求可能无结果，但通常第一次请求能正常返回数据。不影响其他引擎。

解决方案： 使用 Google 或 Brave 作为替代通用引擎。

5.3 端口被占用

启动失败 
OSError: [Errno 98] Address already in use

原因： 上次启动的进程未正常退出。

解决：

pkill -f waitress # 杀掉旧进程 simplexng # 重新启动

5.4 Cookie 验证错误

已修复 
ValidationException: Invalid value: "zh-CN"

原因： 浏览器存储了旧版本的 locale=zh-CN Cookie，当前语言列表中不存在该值。

修复： 已修改 preferences.py，无效 Cookie 值会被静默忽略，保留默认值。或清除浏览器 Cookie 后重新访问。

5.5 配置文件缺失 limiter.toml

无关紧要 
WARNING: missing config file: .../limiter.toml

限流配置文件仅公网实例需要。内网/个人使用完全不影响功能。

六、高级配置参考

6.1 网络与超时设置

在配置文件的 server: 之后添加：

outgoing:
  request_timeout: 5.0          # 单次请求超时（秒），引擎可覆盖
  max_request_timeout: 15.0     # 最大允许超时
  useragent_suffix: ""          # User-Agent 后缀
  enable_http2: true            # 启用 HTTP/2
  pool_connections: 100         # 连接池连接数
  pool_maxsize: 20              # 连接池最大复用数
  # proxies:                     # 代理配置
  #   all://:
  #     - http://proxy:8080

6.2 限流配置（公网实例）

创建 ~/.config/simplexng/limiter.toml：

[botdetection.ip_limit]
link_token = false         # 是否需要链接令牌

[botdetection.ip_lists]
pass_secret = false
pass = []
block = []

并设置 server.limiter: true、redis.url: "redis://localhost:6379/0"。

6.3 自定义外观

general:
  instance_name: "我的搜索"    # 页面标题
  debug: false                  # 调试模式

ui:
  static_use_hash: false        # 静态资源缓存
  default_theme: simple         # 主题
  default_locale: zh-Hans-CN    # 默认语言

6.4 添加新引擎

在 engines: 列表中添加：

  - name: wikipedia
    engine: wikipedia
    shortcut: wp
    timeout: 3.0
    disabled: false

可用引擎列表：~/.local/share/uv/tools/simplexng/lib/python3.12/site-packages/simplexng/_vendor/searx/engines/

6.5 还原补丁（如需升级）

如果升级 SimpleXNG 版本，已修改的源码文件会被 uv 覆盖，需要重新应用补丁：

  • simplexng.py — CLI 默认值从配置文件读取
  • preferences.py — Cookie 验证异常降级
