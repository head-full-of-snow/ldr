Is Grep All You Need? How Agent Harnesses Reshape Agentic Search
📄 arXiv:2605.15184 📅 2026-05-14 🏷️ cs.CL 👤 Sahil Sen, Akhil Kasturi, Elias Lumer, Anmol Gulati, Vamse Kumar Subbiah
因 (问题/动机)
尽管RAG在智能体搜索系统中被广泛采用，但现有文献缺乏对检索策略选择与智能体架构、工具调用范式之间交互关系的系统性比较。工具输出如何呈现给模型、以及当搜索必须应对更多无关文本时性能如何变化，这些实际维度在智能体循环中仍未被充分探索。
法 (方法)
设计了两阶段实验：(1) 使用自定义智能体框架Chronos和提供商原生CLI工具（Claude Code、Codex、Gemini CLI）在LongMemEval的116个问题上比较grep与向量检索，分别测试内联和基于文件的工具结果呈现方式；(2) 在逐步增加不相关对话历史的情况下比较grep-only和vector-only检索，模拟更多干扰场景。
果 (结果)
在Chronos和各提供商CLI中，grep通常比向量检索获得更高的准确率；同时整体得分强烈依赖于所使用的智能体框架和工具调用风格，即使底层对话数据完全相同。
🔗 arxiv.org/abs/2605.15184

Falkor-IRAC: Graph-Constrained Generation for Verified Legal Reasoning in Indian Judicial AI
📄 arXiv:2605.14665 📅 2026-05-14 🏷️ cs.AI cs.CL cs.IR 👤 Joy Bose
因 (问题/动机)
法律推理不是语义相似性搜索。传统基于向量的RAG无法忠实表征判例传播、程序状态转换和法规约束推理等符号推理属性。幻觉判例、过时法规引用和不支持的推理链仍然是LLM法律AI的持续性失败模式，在印度等高案件量司法管辖区对司法公正造成真实后果。
法 (方法)
提出Falkor-IRAC框架，基于IRAC（争议焦点、规则、分析、结论）知识图谱进行图约束生成。将印度最高法院和高等法院判决摄入为IRAC节点结构，存储在FalkorDB中供低延迟智能体遍历。推理时引入验证者智能体（Verifier Agent），仅当可追踪到有效支撑路径时才接受LLM生成答案，同时将教义冲突作为一等输出进行检测。
果 (结果)
在51个最高法院判决的概念验证语料上，验证者智能体正确验证了已完成查询的引用并正确拒绝了伪造引用。使用图原生指标（引用接地准确率、路径有效性率、幻觉判例率、冲突检测率）进行评估，证明了该框架在法律推理中的可靠性。
🔗 arxiv.org/abs/2605.14665

RS-Claw: Progressive Active Tool Exploration via Hierarchical Skill Trees for Remote Sensing Agents
📄 arXiv:2605.13391 📅 2026-05-13 🏷️ cs.AI 👤 Liangtian Liu, Zeyuan Wang, Ziyu Li, Kai Ouyang 等
因 (问题/动机)
在多源异构遥感工具生态中，现有遥感智能体采用被动工具选择范式（全量注册或RAG检索），面临内在局限：全量工具注册在长任务中触发上下文空间不足，而RAG检索可能在关键步骤中遗漏重要工具。被动机制无法在整个任务推理过程中动态平衡"上下文负载"和"工具集完整性"。
法 (方法)
提出RS-Claw架构，重新定义工具选择——智能体应作为工具空间的主动探索者。利用技能封装技术对工具描述进行层次化结构化，使智能体执行按需序贯决策：先通过工具摘要选择相关技能分支，再动态加载详细描述，最终实现精确调用。这种主动范式显著释放了智能体的上下文空间。
果 (结果)
在Earth-Bench基准上的系统实验表明，RS-Claw的主动探索机制有效过滤语义噪声并大幅释放推理空间，实现高达86%的输入token压缩率，在复杂推理评估中全面超越现有Flat和RAG基线。
🔗 arxiv.org/abs/2605.13391

OpenAaaS: An Open Agent-as-a-Service Framework for Distributed Materials-Informatics Research
📄 arXiv:2605.13618 📅 2026-05-13 🏷️ cs.AI cond-mat.mtrl-sci 👤 Peng Kang, Bixuan Li, Xiaoya Huang 等
因 (问题/动机)
材料信息学面临"最后一公里"问题：尽管拥有世界级的LLM模型和海量材料数据仓库，却缺乏跨机构安全组合这些能力的组织基础设施。结构功能材料开发仍以长期迭代、机理复杂和高领域专业需求为特征，超出了单体智能体系统和传统中心化平台的能力。
法 (方法)
提出OpenAaaS——开源分层分布式Agent-as-a-Service框架，基于"代码流动，数据不动"原则。主智能体在不直接访问子智能体数据的情况下规划和分解研究任务；子智能体作为近数据执行节点保持对本地数据集和专有算法的完全主权。框架集成AlphaAgent（基于RAG的材料文献分析执行器），实现跨尺度跨域安全集成。
果 (结果)
AlphaAgent在深度分析问题上达到4.66/5.0评分（对比单次RAG基线），超大规模六元高熵合金描述符数据库服务证明了在严格数据主权约束下的安全近数据执行。为下一代材料智能设计平台建立了可扩展的基础。
🔗 arxiv.org/abs/2605.13618 🔗 GitHub

ProtoMedAgent: Multimodal Clinical Interpretability via Privacy-Aware Agentic Workflows
📄 arXiv:2605.14113 📅 2026-05-13 🏷️ cs.CV cs.AI cs.LG 👤 Alvaro Lopez Pellicer, Plamen Angelov, Marwan Bukhari 等
因 (问题/动机)
可解释原型网络虽能为临床诊断提供引人注目的基于案例的推理，但其原始连续输出缺乏医学文档所需的语义结构。通过标准RAG桥接此差距时频繁触发"检索谄媚"（retrieval sycophancy）——LLM会编造事后合理化来对齐视觉预测，导致临床报告不可靠。
法 (方法)
提出ProtoMedAgent框架，将多模态临床报告形式化为在严格神经符号瓶颈上的迭代、零梯度测试时优化问题。在冻结的原型骨干上，将潜在视觉和表格特征蒸馏为离散语义记忆。在线生成受精确集合论差分和反思性Scribe-Critic循环约束，数学上排除了不支持的叙事声明。引入受k-匿名性和l-多样性控制的语义隐私门。
果 (结果)
在4,160名患者临床队列上，ProtoMedAgent达到91.2%的比较集忠实度，从根本上超越了标准RAG的46.2%。同时利用绑定l-多样性相变系统性地将伪影级成员推理风险绝对降低了9.8%。
🔗 arxiv.org/abs/2605.14113
